# Contributing to TIX

## PR Rules
- One feature per PR
- Reference an issue in your PR
- All tests must pass

## Branch Naming
- `feature/` - New features
- `fix/` - Bug fixes
- `docs/` - Documentation
- `refactor/` - Code refactoring

## Commit Messages
Use conventional commits:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `test:` Adding tests
- `chore:` Maintenance
